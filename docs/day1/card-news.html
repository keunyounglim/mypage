Why — 왜 Claude Code인가
기존 AI의 벽

ChatGPT, Claude.ai를 써봤다면 이 경험이 있을 겁니다.

"내 파일 수정해줘" → "파일 내용을 붙여넣어 주세요"
"이 폴더 정리해줘" → "어떤 파일이 있는지 알려주세요"
"계속 진행해줘" → (새 대화, 이전 맥락 증발)

AI가 아무리 똑똑해도, 당신의 컴퓨터를 직접 볼 수 없기 때문입니다.

What — 핵심 개념 1
에이전트 vs 채팅
기존 AI 채팅 Claude Code (에이전트)
텍스트만 주고받음 파일을 직접 읽고, 쓰고, 수정함
매 대화가 독립적 프로젝트 전체를 기억하며 일함
"알려주세요"가 필요 스스로 파악하고 실행함
결과를 직접 적용해야 함 실행까지 완료하고 확인도 함

What — 핵심 개념 2
컨텍스트 윈도우

AI가 한 번에 기억할 수 있는 정보량입니다. 새 직원 비유로 이해해 보세요.

  • 소 메모장 한 쪽 분량만 기억. 5분 전 대화도 잊을 수 있음.
  • 대 두꺼운 책 한 권을 통째로 기억. 긴 프로젝트 전체를 이해하며 일함.
Claude Code는 컨텍스트가 매우 큽니다.
CLAUDE.md로 컨텍스트를 보존하면 다음 세션에도 기억합니다. (Day 3)
